Mastering React Hooks: Panduan Komprehensif

Dari useState hingga custom hooks - panduan lengkap React Hooks

Pengantar

React Hooks telah merevolusi cara kita menulis komponen React. Sejak diperkenalkan di React 16.8, Hooks memungkinkan kita menggunakan state dan fitur React lainnya tanpa menulis class component. Panduan ini akan membahas semua yang perlu Anda ketahui tentang React Hooks.

Apa itu React Hooks?

Hooks adalah fungsi yang memungkinkan Anda "hook into" React state dan lifecycle features dari function components. Sebelum Hooks, kita harus menggunakan class components untuk menggunakan state dan lifecycle methods.

1. useState - State Management

Hook paling dasar untuk menambahkan state ke function component.

import { useState } from 'react';

function Counter() {
    // Declare state variable
    const [count, setCount] = useState(0);
    
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
        </div>
    );
}

// Multiple state variables
function Form() {
    const [name, setName] = useState('');
    const [email, setEmail] = useState('');
    const [age, setAge] = useState(0);
    
    return (
        // ...
    );
}

// State dengan object
function User() {
    const [user, setUser] = useState({
        name: '',
        email: '',
        age: 0
    });
    
    const updateName = (newName) => {
        setUser(prev => ({
            ...prev,
            name: newName
        }));
    };
}

2. useEffect - Side Effects

Menangani side effects seperti data fetching, subscriptions, atau manual DOM manipulation.

import { useState, useEffect } from 'react';

// Basic useEffect
function DataFetcher() {
    const [data, setData] = useState(null);
    
    useEffect(() => {
        // Runs after every render
        console.log('Component rendered');
    });
    
    // With dependency array - runs once on mount
    useEffect(() => {
        fetch('/api/data')
            .then(res => res.json())
            .then(data => setData(data));
    }, []); // Empty array = run once
    
    // With dependencies
    useEffect(() => {
        // Runs when 'id' changes
        fetchUser(id);
    }, [id]);
    
    // With cleanup
    useEffect(() => {
        const subscription = api.subscribe();
        
        return () => {
            // Cleanup function
            subscription.unsubscribe();
        };
    }, []);
}

3. useContext - Context API

Mengakses context tanpa menggunakan Consumer component.

import { createContext, useContext } from 'react';

// Create context
const ThemeContext = createContext('light');

// Provider
function App() {
    return (
        <ThemeContext.Provider value="dark">
            <Toolbar />
        </ThemeContext.Provider>
    );
}

// Consumer dengan useContext
function ThemedButton() {
    const theme = useContext(ThemeContext);
    
    return (
        <button className={theme}>
            Themed Button
        </button>
    );
}

4. useReducer - Complex State Logic

Alternatif useState untuk state logic yang kompleks.

import { useReducer } from 'react';

// Reducer function
function counterReducer(state, action) {
    switch (action.type) {
        case 'increment':
            return { count: state.count + 1 };
        case 'decrement':
            return { count: state.count - 1 };
        case 'reset':
            return { count: 0 };
        default:
            return state;
    }
}

function Counter() {
    const [state, dispatch] = useReducer(
        counterReducer, 
        { count: 0 }
    );
    
    return (
        <div>
            <p>Count: {state.count}</p>
            <button onClick={() => dispatch({ type: 'increment' })}>
                +
            </button>
            <button onClick={() => dispatch({ type: 'decrement' })}>
                -
            </button>
            <button onClick={() => dispatch({ type: 'reset' })}>
                Reset
            </button>
        </div>
    );
}

5. useRef - Mutable References

Menyimpan mutable value yang tidak trigger re-render.

import { useRef, useEffect } from 'react';

function TextInput() {
    const inputRef = useRef(null);
    
    useEffect(() => {
        // Focus input on mount
        inputRef.current.focus();
    }, []);
    
    return (
        <input ref={inputRef} type="text" />
    );
}

// Store previous value
function Counter() {
    const [count, setCount] = useState(0);
    const prevCountRef = useRef();
    
    useEffect(() => {
        prevCountRef.current = count;
    });
    
    const prevCount = prevCountRef.current;
    
    return (
        <div>
            <p>Current: {count}</p>
            <p>Previous: {prevCount}</p>
        </div>
    );
}

6. useMemo - Memoization

Memoize expensive calculations.

import { useMemo, useState } from 'react';

function ExpensiveComponent({ items }) {
    const [filter, setFilter] = useState('');
    
    // Expensive calculation
    const filteredItems = useMemo(() => {
        console.log('Filtering items...');
        return items.filter(item => 
            item.name.includes(filter)
        );
    }, [items, filter]); // Only recalculate when these change
    
    return (
        <div>
            <input 
                value={filter}
                onChange={e => setFilter(e.target.value)}
            />
            {filteredItems.map(item => (
                <div key={item.id}>{item.name}</div>
            ))}
        </div>
    );
}

7. useCallback - Memoize Functions

Memoize callback functions untuk prevent unnecessary re-renders.

import { useCallback, memo } from 'react';

// Child component
const Child = memo(({ onClick }) => {
    console.log('Child rendered');
    return <button onClick={onClick}>Click</button>;
});

// Parent component
function Parent() {
    const [count, setCount] = useState(0);
    
    // Without useCallback - new function on every render
    // const handleClick = () => {
    //     console.log('Clicked');
    // };
    
    // With useCallback - same function reference
    const handleClick = useCallback(() => {
        console.log('Clicked');
    }, []); // Dependencies
    
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
            <Child onClick={handleClick} />
        </div>
    );
}

8. Custom Hooks

Membuat hooks sendiri untuk reusable logic.

// Custom hook untuk fetch data
function useFetch(url) {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);
    
    useEffect(() => {
        setLoading(true);
        fetch(url)
            .then(res => res.json())
            .then(data => {
                setData(data);
                setLoading(false);
            })
            .catch(err => {
                setError(err);
                setLoading(false);
            });
    }, [url]);
    
    return { data, loading, error };
}

// Usage
function UserProfile({ userId }) {
    const { data, loading, error } = useFetch(`/api/users/${userId}`);
    
    if (loading) return <div>Loading...</div>;
    if (error) return <div>Error: {error.message}</div>;
    
    return <div>{data.name}</div>;
}

// Custom hook untuk localStorage
function useLocalStorage(key, initialValue) {
    const [value, setValue] = useState(() => {
        const saved = localStorage.getItem(key);
        return saved ? JSON.parse(saved) : initialValue;
    });
    
    useEffect(() => {
        localStorage.setItem(key, JSON.stringify(value));
    }, [key, value]);
    
    return [value, setValue];
}

Rules of Hooks

⚠️ Aturan Penting:

  • Only call Hooks at the top level: Jangan panggil Hooks di dalam loops, conditions, atau nested functions
  • Only call Hooks from React functions: Function components atau custom Hooks
  • Custom Hooks harus dimulai dengan "use": Convention untuk mudah identify

Best Practices

  • Gunakan useState untuk simple state, useReducer untuk complex state
  • Dependency array di useEffect harus complete dan accurate
  • Extract reusable logic ke custom Hooks
  • Gunakan useMemo dan useCallback untuk optimization, tapi jangan premature optimization
  • Keep components small dan focused

Common Patterns

1. Fetch Data Pattern

function DataComponent() {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);
    
    useEffect(() => {
        let cancelled = false;
        
        setLoading(true);
        fetchData()
            .then(result => {
                if (!cancelled) {
                    setData(result);
                    setLoading(false);
                }
            })
            .catch(err => {
                if (!cancelled) {
                    setError(err);
                    setLoading(false);
                }
            });
        
        return () => {
            cancelled = true;
        };
    }, []);
}

2. Debounce Pattern

function useDebounce(value, delay) {
    const [debouncedValue, setDebouncedValue] = useState(value);
    
    useEffect(() => {
        const timer = setTimeout(() => {
            setDebouncedValue(value);
        }, delay);
        
        return () => clearTimeout(timer);
    }, [value, delay]);
    
    return debouncedValue;
}

Kesimpulan

React Hooks adalah game-changer dalam React development. Mereka membuat code lebih clean, reusable, dan mudah dipahami. Dengan menguasai Hooks - dari basic useState dan useEffect hingga advanced custom Hooks - Anda dapat menulis React applications yang lebih efficient dan maintainable.

Practice adalah kunci! Mulai refactor class components Anda ke function components dengan Hooks, dan coba buat custom Hooks untuk logic yang reusable.
